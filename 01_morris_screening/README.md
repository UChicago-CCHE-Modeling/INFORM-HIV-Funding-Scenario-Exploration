# 01 — Morris Screening

Morris elementary-effects screening to identify which model parameters most
strongly influence the epidemic outcomes of interest. This is the first stage of
the analysis: a global, one-at-a-time (OAT) sensitivity screen used to rank the
21 candidate parameters before more expensive downstream analyses.

## Method

The Morris method perturbs one parameter at a time along a set of random
trajectories through the input space and records the resulting change in each
output ("elementary effect"). Averaging these effects over trajectories yields,
per parameter and per output:

- **mu** — mean elementary effect (signed; direction of influence)
- **mu\*** — mean of the absolute elementary effects (overall influence / importance)
- **sigma** — standard deviation of the elementary effects (non-linear and
  interaction effects)

The design uses **r = 10** trajectories over **21** parameters, giving
`10 * (21 + 1) = 220` simulation runs.

Inputs are screened on a normalized `[0, 1]` scale and outputs are
z-standardized, so that mu\* and sigma are directly comparable across parameters
and across quantities of interest.

## Quantities of interest (QOIs)

Each simulation run reports four summary outcomes:

| QOI | Meaning |
|-----|---------|
| `avg_incidence_10yr`  | mean HIV incidence over a 10-year horizon |
| `avg_incidence_5yr`   | mean HIV incidence over a 5-year horizon |
| `avg_prevalence_10yr` | mean HIV prevalence over a 10-year horizon |
| `avg_prevalence_5yr`  | mean HIV prevalence over a 5-year horizon |

## Contents

### `data/`
- **`sensitivity_params.csv`** — the 21 screened parameters with a point
  `estimate` and lower/upper bounds (`lb`, `ub`) defining the screening range.
- **`morris_design.rds`** — saved `sensitivity::morris()` design objects:
  - `morris_design` — design on the natural parameter scale (bounds from
    `sensitivity_params.csv`).
  - `morris_design_01` — the same design on the normalized `[0, 1]` scale; this
    is the object used by the analysis script.
- **`morris_design.csv`** — the 220 design points on the natural parameter scale
  (human-readable export of the design that was fed to the simulator).
- **`morris_design_outputs.csv`** — the design points with the four simulated
  QOIs appended (one row per run). This is the main input to the script.
- **`upf_json.txt`** — the "universal parameter file": one JSON record per
  simulation run, as consumed by the epidemic model. Retained for provenance;
  not read by `morris.R`.

### `script/`
- **`morris.R`** — computes the Morris indices for every parameter/QOI pair and
  writes the diagnostic plots. Run it with `script/` as the working directory:

  ```sh
  cd 01_morris_screening/script
  Rscript morris.R
  ```

  Requires the R packages `data.table`, `ggplot2`, and `sensitivity`.

### `output/`
Generated by `morris.R`:

| File | Description |
|------|-------------|
| `morris_scatter_plot.png`                 | mu\* vs sigma scatter, faceted by QOI |
| `morris_mustar_heatmap_std.png`           | mu\* heatmap (parameter × QOI) |
| `morris_sigma_heatmap.png`                | sigma heatmap (parameter × QOI) |
| `morris_musigma_heatmap.png`              | mu\* × sigma heatmap (parameter × QOI) |
| `morris_ranked_heatmap_single_plot_std.png` | parameters ranked by mu\* within each QOI |
